Intro

In the Admin Settings, you can handle many administrative functions in Domo, including managing users and groups, controlling user access to content, and changing your organization's name and logo that appear in Domo.

This article provides information about the Admin Settings in the following topics:

  • Access the Admin Settings
  • Navigate the Admin Settings

Access the Admin Settings

In the navigation header, select More > Admin. The Admin Settings screen displays. Depending on your level of Domo access, your Admin Settings may look different than the example below.

Navigate the Admin Settings

The Admin Settings screen and the left side rail, which you can expand by selecting the  hamburger menu, both contain the menu items listed below, depending on your Domo security settings.

Authentication

The Authentication menu contains the following sections:

  • Authentication
  • OpenID Connect (SSO)
  • SAML (SSO)
  • Access tokens
  • API Clients

Authentication

The Authentication menu contains options for authenticating in Domo, including password criteria, username autocomplete, and multi-factor authentication. For more information about these topics, see the following articles:

  • Setting Password Requirements
  • Changing Your Password
  • Disabling Username Autocomplete in the Login Screen
  • Turning On Multi-Factor Authentication

OpenID Connect (SSO)

The OpenID Connect (SSO) section lets you configure SSO using OpenID Connect. For more information, see Enabling SSO with OpenID Connect.

SAML (SSO)

The SAML (SSO) section lets you configure Single Sign-On (SSO) for your Domo instance. For more information, see Understanding and Configuring Domo Single Sign-On.

Access Tokens

The Access tokens section lets you view a list of access tokens that have been issued. For more information, see Manage Access Tokens.

Network Security

The Network security menu contains the following sections:

  • Invite authorized domains
  • Custom Apps authorized domains
  • Embed authorized domains
  • IP Address Whitelist

Invite Authorized Domains

The Invite authorized domains section lets you specify email domains for those who can be invited to Domo. For more information, see Specifying Authorized Domains for Invited Users.

Custom Apps Authorized Domains

The Custom Apps authorized domains section lets you specify the domains allowed for navigation from a Custom App. If this is left blank, it allows navigation to all domains from a Custom App.

Embed Authorized Domains

The Embed authorized domains section allows you to set a bounce page. Depending on browser type and user settings, the Domo cookie may be rejected when embedding Domo in an iFrame. For example, the Safari browser requires a user to either visit a Domo site first or change their privacy settings in order to load Domo in an iFrame. Learn more about Safari here.

To support these browsers, Domo provides a bounce page. This bounce page automatically loads a Domo page that sets a cookie, and then returns the user to the referring site. For security reasons, this bounce page only works when the user’s browser originates from certain sites. Wild cards in the form of an asterisk “*” are allowed as a placeholder for the subdomain or path.

IP Address Whitelist

The IP Address Whitelist section lets you place specific IP addresses on an allowlist in Domo. For more information, see Whitelisting IP Addresses in Domo.

Features

The Features menu contains the following sections. Goals and Cloud Amplifier have their own documentation, linked below. 

  • Certified Content
  • Approvals
  • Slideshows
  • Buzz
  • Goals
  • Cloud Amplifier
  • Connectors

Certified Content

In the Certified Content section, you can see a list of all Domo cards or DataSets that have been certified, are awaiting certification, or have an expired certification. You can also modify the fields and approvers in the Card or DataSet certification request form. You must have an Admin security profile to see this section.

For more information, see Certifying Cards and DataSets.

Slideshows

In the Slideshows section, you can see the names of the slideshows in your instance, the card slides in each slideshow, along with any published URL and access code for each slideshow.

You can filter the list of slideshows by entering text in the Search field.

For more information about slideshows, see Sharing Content Using Slideshows. 

Buzz

In the Buzz section, you can configure global options for Buzz in your Domo instance. You can also see activity indicators for the number of Buzz messages sent and users for this week compared to last week.

The options you can configure in this section include the following:

  • Enable company-wide General channel. When this box is checked, a "General" conversation is available for all Buzz users in your instance.

  • Default all conversations around Domo data to private. When this box is checked, the conversations on Domo data are set to private.

  • Allow all users to invite Social users. When this box is checked, anyone in your Domo can invite users to Domo as Social users. For more information, see Inviting Others to Join Domo.

The Buzz section displays in your Admin Settings only if you have an Admin default security role or a custom role with the Edit Conversations and Messages grant enabled. For more information about default security roles, see Managing Roles.

For more information about Buzz, see Buzz Layout.

Connectors

The Connectors section allows you to grant and revoke access to power up Connector DataSets. For more information, see Connector Permissioning in Domo.

More

The More menu contains the following sections:

  • Manage easy links
  • Exporting data
  • Alert Actions
  • Tool Downloads

Manage Easy Links

The Manage easy links section lets you share links with team members so they can start collaborating with you over Buzz. For more information, see Inviting Others to Join Domo.

Exporting Data

The Exporting data section lets you enable or disable Excel formulas before exporting Domo data.

Alert Actions

The Alert Actions section allows you to enable Alert actions, which allow users to use an Alert trigger event to automatically take action in a third-party application via webhook or API. These actions may transmit data from the Card or DataSet that the Alert is set on to the third-party application. Disabling Alert actions prevents users from creating Alert actions and will disable any currently functioning actions in Domo.

Tool Downloads

In the Tool Downloads section, a panel displays with information about tools you can download. Currently, you can choose between the following tools:

  • Workbench 5
  • Workbench 5.1
  • Domo ODBC Driver 1.0
  • Domo PowerPoint Plugin
  • Domo Excel Plugin
  • Domo Word plug-in
  • Domo CLI
  • Domo Google Sheets add-on
