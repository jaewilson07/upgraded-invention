Required Grants

The following grants give user-specific rights to Queues within your Domo environment:

  • Manage Task Queues — Allows you to perform any action on any queue in the instance. This grant is for superusers, admins, or anyone who helps manage queues.
  • Create Task Queues — Allows you to create new task queues. This grant is for any user who needs the ability to create and add new queues to the workflow repository in your instance.

Contact your Domo accout team to receive either of these grants for your instance. For information about the available permissions for non-instance admin users, see the Queue Permissions section of this article.

Access Queues

You can access Queues from the side navigation of any page in Workflows, Code Engine, or Forms. From the side navigation, select  Queues.
The Queues home page displays.

 
Note: If  Queues does not display in the side navigation, contact your Domo account team for assistance.

Home Page

The Queues home page is the entry point for viewing and accessing the various features of the tool. On the home page, you can see the list of queues you have access to, task counts and volumes for the queues you have access to, as well as your recent activity and a watchlist of tasks you are currently monitoring. At the top of the page, Live Updates are toggled by default. If you turn off Live Updates, you must refresh the page to see the updated counts and volumes.

Learn about the sections of the home page below.

Queues Panel

The Queues panel displays a list of all the queues to which you have access. Select a queue in the panel to navigate to the task list for that queue.

 In the panel, a count of all the open tasks in the queue display next to the queue name.

 

Note: Live Updates to everything on the home page are toggled by default. If you turn off Live Updates, you must refresh the page to see the current task counts.

Tiles

Tiles at the top of the home page provide counts and quick links to filtered task lists. Selecting one of the tiles, described below, opens the Task List with one or more predefined Filters.

  • Open Tasks — Filters the Task List to those with an Open status
  • My Tasks — Filters the Task List to those assigned to you with an Open status
  • Completed Today — Filters the Task List to those you have placed in a Completed status during the current day

Open Task Count

The Open Task Count graph updates throughout the day as tasks are added and completed in your queues. The count refreshes every 15 minutes.

There are two views of the graph:

  • All Tasks — Displays the number of open tasks and how many of them have been assigned to you throughout the day.
  • Tasks per Queue — Displays the number of open tasks and how many are in each queue throughout the day.

My Recent Activity

You can view a list of the last ten tasks you interacted with in the My Recent Activity tab of the home page. You must make a change to a task for it to display in the list.
Select any row in the list to navigate to that task.

Watchlist

In the Watchlist tab on the home page, you can view the assignee and status of tasks you need to monitor. Selecting a task in the Watchlist takes you to the task's details.

You can add a task to the Watchlist only while viewing the task details. If you want to remove a task from the Watchlist, select the flag icon in the Watchlist tab to remove it.

Queue Manager

You can view the Queue Manager by selecting  Manage Queues in the Queues panel on the home page.

 

In the Queue Manager, users see a list of all the queues they have access to. Admins see a list of all queues in the instance and can add or remove queues and control access.

Use the search, filter, and sorting controls to locate a queue or arrange the list in a different order. By default, the queue list is sorted by most recently updated to least recently updated.

Create a Queue

Queues admins can follow these steps to create a queue:

  1. In the Queue Manager, select + New Queue.

     

    ​​​​​The New Queue modal displays.

     

  2. In the modal, fill out the Queue Name and Queue Description fields.
  3. Select New Queue to save the queue and display it in the Queue Manager.

Queue Permissions

To manage permissions to a queue, admins can select the  kebab menu at the end of its row in the Queue Manager.

 

Selecting Manage permissions in the  kebab menu launches the permission modal (labeled Share Access) for the specific queue.

In the permissions modal, search for the user(s) or group(s) whose access you want to update. Users and groups with some level of access appear in the modal above those without any access.


Use the access dropdowns to make changes to access. These permissions are granular so you can give users or groups access only to certain actions in a queue or task.

The following list describes the various access levels:

  • Admin — Allows a user to perform any action on a specific queue or task in that queue
  • Read — Allows a user view-only access to queue settings in Queue Manager
  • Write — Allows a user edit access to a queue
  • Share — Allows a user to grant another user equal or lesser access to a queue 
  • Create Content — Allows a user to create a task to display in the queue
  • Read Content — Allows a user view-only access to a task and the queue count
  • Update Content — Allows a user to edit and submit a task
  • Void Content — Allows a user to void a task, which prevents manual updates from occurring

Queue Notifications

Queue notifications settings allow you to receive near real-time notification of specific events occurring in queues, based on your preferences. Note that if a queue is archived, notifications are no longer sent.

Any user can follow these steps to access the queue notification settings for the queues to which they have access:

  1. In the Queue Manager, select the  kebab menu for a queue to manage its notifications.
  2. In the menu, select Manage notifications.

     

  3. On the Manage notifications page, toggle Allow notifications from this Queue.

     

  4. Select which notifications for your tasks and which notifications for all tasks you want to receive and by what medium—push notification to your computer, email, or SMS.

Assignee Security

The Assignee Security setting restricts task visibility to assigned users and/or their groups. Queues administrators can enable/disable this setting for each queue.

When the setting is enabled, users without Admin permission will only see tasks assigned to them or their groups. All group members can see tasks assigned to the group. Users with Admin permission for the queue will continue to see all tasks.

Admins can follow these steps to enable Assignee Security for a queue:

  1. In the Queue Manager, select the kebab menu for the queue for which you want to manage notifications.
  2. In the menu, select Assignee Security to display the modal.

     

  3. In the modal, check the box to Enable Assignee Security and select Save.

Archiving Queues

Users with Admin permission to a queue can archive it and make it inactive for end users. However, users can still access archived queues and retrieve completed tasks that reside within them. 

When a queue is archived, no tasks can be generated in it or transferred to it. In addition, all open, unresolved tasks are either moved to another queue or updated to Void status as part of the archiving process. All previously completed tasks remain in the archived queue and can be retrieved as needed for audit purposes. Learn more about Task Status below.

Follow these steps to archive queues:

  1. In the Queue Manager, select the kebab menu for the queue you want to archive.
  2. In the menu, select Archive Queue to open the archive modal. The modal displays a list of all tasks in the queue.

     

  3. In the dropdown, select Transfer Tasks to move open tasks to another queue or Void Tasks to cancel them.
  4. Select Save and Archive.

After tasks are archived, a success message displays and an "Archived" label displays beside the queue name. All users can still retrieve completed tasks in archived queues.

 

 
Important: Workflows with User Tasks configured with an archived queue will fail on every newly executed instance. Queues admins must coordinate with the workflow owners to make sure that a new version of the workflow is configured to reference another queue before the queue is archived.

Activate Archived Queues

Follow these steps to activate an archived queue:

  1. In the Queue Manager, locate the archived queue you want to activate and select thekebab menu.
  2. Select Activate Queue.

Task List

You can access the Task List by selecting a queue or tile from the home page.

Selecting a tile always takes you to the Task List, but the results in the list are filtered based on the tile you select. For example, if you select the My Tasks tile, the Task List opens, filtered to show the tasks assigned to you with a status of Open across all your queues. To learn more, see Tiles.

If you select a queue from the Queues panel on the home page, the Task List displays the open tasks for that queue.

Each row in the Task List represents a single task. Learn about tasks below. 

A results count displays under the search bar so you can see the number of total tasks (depending on Filters).

The columns in the Task List provide metadata for each task, as follows:

  • Task ID — The unique identifier for the task. It consists of the task creation date and an alphanumeric character set.
  • Task Identifiers — Tags or field values to help identify and filter tasks. Learn about creating Task Identifiers in a Form.
  • Form — The form name associated with the task.
  • Queue — The assigned queue for the task.
  • Assignee — The name and avatar of the person the task is assigned to.
  • Created By — The name and avatar of the person who created the task.
  • Status — The status of the task. The options are Open, Complete, and Void. See Task Status for more information.

Filters

You can update your view of the Task List by changing the Filters. 

 

You can apply multiple Filters to the Task List at the same time, but you can specify only one selection per Filter. The available Filters are described below.

  • Assigned To — Filters the Task List by the assignee. Select a user with this Filter to see that user's assigned tasks. Clearing this Filter displays tasks for all assignees and all unassigned tasks.
  • Completed By — Filters the Task List by the user who completed the task.
  • Completed On — Filters the Task List by the date a task was completed. This can be used with the Status Filter set to Complete. You can narrow the Filter by time period, including Today, Last 7 Days, Last 30 Days, Year to Date, and Lifetime. Clearing this Filter displays tasks completed on any date.
  • Created On — Filters the Task List by task creation date. You can narrow the Filter by time period, including Today, Last 7 Days, Last 30 Days, Year to Date, and Lifetime. You can also select Toggle Custom Range to choose the start and end dates between which tasks were created. Clearing this Filter displays tasks created on any date.
  • Form Filter — Filters the Task List by form name associated with the tasks.
  • Queue Filter — Filters the Task List by queue. Selecting a queue from the Queues Panel populates this Filter by default. The list displays options from other queues that are accessible to you. Clearing this Filter displays tasks in all queues.
  • Status Filter — Filters the Task List by status. Most tiles on the home page populate this Filter with Open. You can choose to change the status to Complete or Void. Clearing this Filter shows tasks with any status.

Tasks are generated by a workflow and represent a manual step in that workflow. Each task is associated with a form that needs to be completed by a person. When the task is complete, the workflow progresses to the next step.

Open a task by selecting a row in the Task List. The task displays in the Form Viewer at the right side of the screen. You can interact with the task through various controls, explained below.

Task Action Menu

The task action menu allows users with specific permissions to perform certain actions on the task. Select this menu to access the following options:

  • Void Task — Cancels the task and updates its status to Void. A voided task does not pass any data back to the workflow and places the workflow in a Failed state.
  • Unlock Task — Unlocks the task while it is being edited by another user. Whenever a task is open, it is locked so that other users cannot update it. Admins can unlock a task so more than one user can edit it simultaneously.
  • Transfer Queue — Moves the task from one queue to another. When this option is selected, an admin must specify another queue to send the task to via a modal that displays. The admin can only select other queues that they have access to.
  • Expand — Displays the task in a full-page view. When the task is expanded, the icon changes to Collapse, which you can select to return to the previous view.
  • Close — Closes the task. This action does not save any data entered in the task fields. See Form Tab for how to save changes.

Task Status

Tasks can have one of the following statuses:

Open

All tasks have an Open status when they are created by a workflow. Anyone with Update Content access to the queue in which the task resides can update an Open task, regardless of whether they are assigned to the task.

Completed

You must submit a task for it to have a Completed status. After a task has been submitted and has a Completed status, you cannot edit it or change it back to an Open status.

Void

Only an admin user can apply the Void status to a task. A voided task means that you do not want to submit any data, and the associated workflow that created the task is placed in a failed state. After a task is voided, you cannot edit it or change it back to an Open status.

Tabs

The Form Viewer contains tabs for each task to help you enter and review data for the associated form. The tabs are located below the task header.

Form Tab

The Form tab displays editable fields for the form. You can also save and submit forms from this tab.

The options available in this tab are described below:

Previous and Next

If the form has multiple pages, you can navigate through them using Previous and Next. Selecting Next displays the following page in the form, and selecting Previous displays the prior page.

Save

After entering data in the fields, you can save the form without submitting it by selecting Save. This saves all the data entered in the task and keeps the form in an Open status. You can then close the task and return to the form in its most recently saved state.

To learn about task statuses, see Task Status below.

Submit

The option to Submit is enabled only when you fill in all the required fields in the task. Selecting this option submits the data you entered in the fields and updates the task to Completed status. The workflow that created the task consumes any data marked as output in the form configuration. After you submit a task, you cannot make updates or change the status back to Pending.

To learn about task statuses, see Task Status below.

 

Note: Anyone with the Update Content permission for the queue containing a task in Open status can update the task, regardless of whether they are assigned to the task.

History Tab

The History tab displays all update events for the task. For every event, you can view the time, the person who made the update, the event type, and a description.

Back to top

Comments Tab

The Comments tab displays comments that users have added to the task. These can be additional notes, communication with other users, or general information.

 

Back to top

Related Tasks Tab

The Related Tasks tab displays metadata for additional tasks generated by the same workflow instance. This feature provides a quick way to locate and view all the sibling tasks created by the same workflow executed instance. Each task displayed in the Related Tasks tab can be selected, allowing the user to jump to other tasks with relevant details to review or update.

To view details of a related task, select the Task ID below its name, and the application redirects you to that task.

 

 

Notes:

  • Users can only see other tasks to which they have access.

  • This feature does not allow manually linking a task to another task.
